# 08-gltf-zoo — Milestone 6's "done when"

**Import → cook → load → render real glTF models with textures, through the whole offline→runtime
asset pipeline.** This is the sample the whole milestone was building toward: a small turntable
gallery of three hand-authored glTF models on the mipmapped-checker floor, each cooked to `RMA1` by
the `rime` CLI, loaded back through `engine/assets`, uploaded via the GPU asset bridge, and drawn with
PBR + tonemap through the render graph. It closes M6 the way `07-first-light` closed M5 — the
milestone ends in a **runnable proof, not a compile**.

Each model earns its place by exercising one M6 feature end to end (the source assets are described in
[`assets/README.md`](assets/README.md); they are **first-party**, generated by
[`assets/generate.py`](assets/generate.py), so nothing here carries a third-party license):

- **`cube`** — a textured box: **base-color texturing** (a 32-byte position/normal/uv mesh, one
  material, one sRGB albedo).
- **`sphere`** (the hero) — a UV sphere with a genuinely bumpy **normal map** and a varying
  **metallic-roughness** map: **MikkTSpace tangent generation** (the normal map gates it → 48-byte
  tangented vertices), **normal mapping**, and **per-usage colour space** (albedo cooks sRGB; the
  normal/MR maps cook linear — all from one file).
- **`rig`** — a skinned three-joint column shown **CPU-posed** at the mid-point of its `Bend`
  animation: **skinning (AN0)** — a `.rskel` + `.ranim` sampled once by M6.7's CPU sampler and
  linear-blend-skinned into a static posed mesh (the honest preview until GPU palette skinning, AN1).

## What each pipeline stage does

- **cook** — `rime cook assets/ --out <dir>` writes one `manifest.txt` plus the
  `.rmesh`/`.rmat`/`.rtex` (+`.rskel`/`.ranim`) files. A CTest fixture runs it before the headless
  self-check, exactly as a user would.
- **load** — the manifest is the whole index: for every cooked mesh we resolve its material (by the
  `<source>#materialN` link), and each material's texture ids back to cooked files. Meshes + textures
  load **async** on the job system ([`AssetServer`](../../engine/assets), M6.5); materials, skeletons,
  and clips are small fixed records read synchronously. The sample reads **no glTF** — only `RMA1`.
- **GPU** — the [`GpuAssetBridge`](../../docs/adr/0025-gpu-asset-bridge.md) uploads a cooked texture's
  offline mip chain verbatim and swaps a material's magenta placeholder for the real texture on drain.
- **AN0** — a rigged mesh is deformed once on the CPU: sample its clip into a joint palette, then
  linear-blend-skin the vertices into a static posed mesh ([docs/math/skinning.md](../../docs/math/skinning.md)).

## Modes

```
build/dev/bin/gltf_zoo --headless [--cooked <dir>] [--frames 30] [--ppm out.ppm]
build/dev/bin/gltf_zoo --serve    [--cooked <dir>] [--host 0.0.0.0] [--port 9100]
build/dev/bin/gltf_zoo --serve --slow-io 400   # stagger the texture pop-in for the demo
```

- **`--headless`** (CI): renders off-screen and runs **structural pixel asserts** — the scene is lit;
  the base-color texture and the normal map each demonstrably move the shaded image when stripped (a
  control render omits one map and diffs); at least one model exercised the AN0 posed path; and at
  least one texture reached the GPU through the bridge. Exit code is the proof. On a machine with no
  Vulkan device it is a **guarded skip** (returns 0) unless `RIME_REQUIRE_VULKAN=1`.
- **`--serve`** streams the live gallery over the **Track S0** stack to the `04-remote-view` client
  (the dev-server → Mac path). `--slow-io <ms>` withholds the GPU upload for a warm-up window so you
  can *watch* the magenta placeholders swap for the real textures — the M6.5 async story, made visible.

## Cook it by hand

The CTest fixture cooks automatically, but to cook into a directory yourself:

```
cargo run --manifest-path tools/Cargo.toml --bin rime -- \
    cook samples/08-gltf-zoo/assets --out /tmp/zoo-cooked
build/dev/bin/gltf_zoo --headless --cooked /tmp/zoo-cooked --ppm /tmp/zoo.ppm
```

Cooking the whole `assets/` **directory** in one invocation produces a single `manifest.txt` that
indexes every model; the loose PNGs under `textures/` are pulled in by each material with the correct
per-usage colour space rather than cooked as standalone textures.
