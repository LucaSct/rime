# Asset test fixtures

Small, self-contained assets used by the M6 asset-pipeline and `engine/assets` tests.

- `quad.gltf` — a two-triangle quad (4 vertices, position/normal/uv, `u16` indices) whose single node
  is translated by `(1, 2, 3)`, so cooking it exercises transform flattening.
- `quad_flat.gltf` — the same quad with **no** `NORMAL` accessor, to exercise the cooker's flat-normal
  derivation.
- `quad.rmesh` — `quad.gltf` **cooked** to RMA1 by `tools/asset-pipeline`. This is the cross-language
  proof artifact: `tools/asset-pipeline/tests/cook_fixture.rs` asserts the cooker still produces these
  bytes, and `tests/assets/fixture_test.cpp` asserts the C++ reader ingests them.
- `checker.png` — a 2×2 sRGB checker (top row white/black, bottom row black/white), the canonical gamma
  fixture. Its top-left texel is white, which pins texture row order across the cook.
- `checker.rtex` — `checker.png` **cooked** to RMA1 (M6.3): the texture cross-language proof. Its 1×1
  mip is sRGB `188` — the gamma-correct average — so `cook_fixture.rs` and `fixture_test.cpp` together
  prove both languages agree on the *gamma-correct offline mip chain*, not just the container.
- `material_quad.gltf` — the M6.4 material fixture: the same unit quad, now with one metallic-roughness
  material whose factors are the distinct values the cross-language material round-trip uses. It
  references `checker.png` as base-color/emissive (sRGB) **and** metallic-roughness/occlusion (linear) —
  exercising per-usage colour space and texture dedup — and `normal.png` as its normal map, which is
  what makes the cooker generate tangents. Not cooked to a golden here: the material *bytes* are already
  pinned by the schema hash + the shared distinct-value record, so `cook_fixture.rs` asserts on the
  structured import (factors, dedup, the tangent gate) instead of brittle golden bytes.
- `normal.png` — a 2×2 flat tangent-space normal map (every texel `(128,128,255)` = `+Z`, "no
  perturbation"), generated by the same stdlib-only script. Used by `material_quad.gltf`.
- `cube.stl` — a 2×2×2 **binary STL** cube (12 triangles, 36 un-indexed soup vertices), emitted with
  the same corner set and winding as the viewer's `make_unit_cube()`. The M6.6 STL source fixture.
- `cube.rmesh` — `cube.stl` **cooked** to RMA1: the STL cross-language proof.
  `tests/viewer/cooked_dogfood_test.cpp` loads both and asserts the cooked mesh index-expands to the
  exact soup the viewer's live STL loader builds (positions bit-exact, normals to a sub-ULP margin),
  with dedup collapsing the 36 soup vertices → 24 unique corners.
- `cube.rsdf` — `cube.stl` **cooked to a signed-distance field** (M10.4a, ADR-0032 §2): the SDF
  cross-language proof. `tools/asset-pipeline/tests/cook_fixture.rs` asserts the cooker still
  produces these bytes; `tests/assets/fixture_test.cpp` asserts the C++ reader ingests them and that
  `sample_mesh_sdf` reproduces the analytic unit-box distance (docs/math/sdf.md §5) at several
  face-interior points. Because `cube.stl`'s vertices are split per face (flat/hard-edge shading —
  see above), the cook reports "not watertight" edges at every hard edge (each edge's two endpoint
  positions are duplicated with a different normal per adjacent face, so they are distinct *vertex
  indices* even though they are the same *point*); this is the cooker degrading honestly exactly as
  designed (docs/math/sdf.md §3/§5), and is why the fixture's own sample points deliberately avoid
  the cube's edges/corners.
- `skinned.gltf` — the M6.7 skeletal fixture: a 2-bone rig (root `A` at the origin, child `B` two
  units along `+X`) skinning a quad, plus one animation `Spin` (A slides `+Y`, B rotates 90° about
  `+Z`, both LINEAR). Its skin deliberately lists joints **child-first** (`B` before `A`), so cooking
  it exercises the importer's topological reorder; the quad carries `JOINTS_0`/`WEIGHTS_0`, so it cooks
  a **skinned** mesh (56-byte stride).
- `skinned.rskel` — `skinned.gltf`'s skin **cooked** to RMA1 (M6.7): the skeleton cross-language proof.
  `cook_fixture.rs` asserts the cooker still emits these bytes; `fixture_test.cpp` asserts the C++
  reader ingests them, reordered parent-first (a non-topological skeleton is rejected, so acceptance is
  the proof), and that sampling the clip against it yields a finite, orthonormal palette.
- `skinned.Spin.ranim` — `skinned.gltf`'s `Spin` animation **cooked** to RMA1 (M6.7): the clip
  cross-language proof, read and sampled by `fixture_test.cpp`.

These glTF files are **hand-authored for Rime** (generated from a small script, embedded buffers as
base64); `checker.png` is generated by a stdlib-only script (no external tool) — none are vendored from
any external collection, so they carry no third-party license; they are covered by the repository's
Apache-2.0.

To regenerate the cooked fixtures after an intentional format change (delete the byproduct
`manifest.txt` and `cook-cache.txt` afterwards — the fixture tests read the `.rmesh`/`.rtex` directly,
not a manifest):

```
cargo run --manifest-path tools/Cargo.toml --bin rime -- \
    cook tests/assets/fixtures/quad.gltf --out tests/assets/fixtures
cargo run --manifest-path tools/Cargo.toml --bin rime -- \
    cook tests/assets/fixtures/checker.png --out tests/assets/fixtures   # sRGB by default
# cube.rmesh: cook to a temp dir and copy the .rmesh back, to avoid the byproduct files here.
cargo run --manifest-path tools/Cargo.toml --bin rime -- \
    cook tests/assets/fixtures/cube.stl --out /tmp/rime-cook && cp /tmp/rime-cook/cube.rmesh tests/assets/fixtures/
# cube.rsdf: same source, the `sdf` subcommand instead of `cook` (writes only the one file, no
# manifest/cache byproducts here since `sdf` — like `fracture` — is not routed through cook_path).
cargo run --manifest-path tools/Cargo.toml --bin rime -- \
    sdf tests/assets/fixtures/cube.stl --out tests/assets/fixtures --name cube
# skinned.rskel / skinned.Spin.ranim: same temp-dir-and-copy (the cook also writes the .rmesh + .rmat).
cargo run --manifest-path tools/Cargo.toml --bin rime -- \
    cook tests/assets/fixtures/skinned.gltf --out /tmp/rime-cook && \
    cp /tmp/rime-cook/skinned.rskel /tmp/rime-cook/skinned.Spin.ranim tests/assets/fixtures/
```
